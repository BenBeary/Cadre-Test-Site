This guide walks you through the four steps to publish blog posts to the CADRE Alumni site. You only need to do this once — after setup, sign in from the post generator and start writing.

1
Accept the collaborator invite

The repo owner will invite you as a collaborator on BenBeary/Cadre-Test-Site. You'll receive:

  • An email titled "[GitHub] @BenBeary has invited you to collaborate"
  • A notification on your GitHub home page

Click View invitation and then Accept invitation. Without this step, the token you generate in Step 2 will not be able to publish posts.

2
Generate a classic Personal Access Token

Open GitHub's token generator and create a classic PAT. The button below pre-fills the form for you.

Generate classic PAT →

On the GitHub page that opens:

  • Confirm only the public_repo checkbox is ticked (it's pre-checked for you). Do not tick anything else — no other scope is needed.
  • Set Expiration to whatever you prefer — 90 days is a reasonable default. You'll need to repeat this step when the token expires.
  • Scroll to the bottom and click Generate token.
Why public_repo and not the full repo scope? The CADRE site is a public repo, and public_repo is the narrowest scope that lets you publish here. The broader repo scope would grant access to all your private repos too — unnecessary for this tool.
3
Copy the token

GitHub will show your new token once. It looks like ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.

⚠ Copy it now. If you navigate away before copying, you'll have to delete it and generate a new one.
4
Paste it into the post generator

Open the post generator, click 🔒 Sign in in the top-right, and paste your token into the input field.

Open the Post Generator →

If your token is valid and you've been added as a collaborator, you'll be signed in and redirected to the admin tools. From there you can write posts, upload images, and publish directly to the site.

?
Why classic and not fine-grained?

GitHub offers two kinds of Personal Access Tokens: fine-grained (newer, more secure) and classic (older, broader scopes).

Fine-grained PATs are restricted to repos owned by the token creator's own account or organization — they cannot target repositories where you are only a collaborator. Since this repo is owned by BenBeary, contributors must use a classic PAT with the repo scope to publish here.

If you own a different repo that uses these tools, you can generate a fine-grained PAT instead using the link on the sign-in modal.
